* Overview

pile.el provides a structured way to display process output in Emacs using magit-section. It creates an organized buffer where command output is shown in collapsible, navigable sections with clear status indicators.

* Features

- Display process output in collapsible magit-section format
- Exit code visualization with color-coded status
- Asynchronous process execution with async-await
- Eshell output filter integration for proper ANSI handling
- Clean, organized buffer management

* Requirements

- Emacs 26.1 or later
- magit-section 2.90 or later
- async-await 1.1 or later

* Installation

** Using leaf

#+begin_src emacs-lisp
(leaf pile
  :ensure t)
#+end_src

** Using use-package

#+begin_src emacs-lisp
(use-package pile
  :ensure t)
#+end_src

** Manual Installation

Clone this repository and add it to your load path:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/pile.el")
(require 'pile)
#+end_src

* Usage

Open the pile buffer:

#+begin_src emacs-lisp
(pile-buffer)
#+end_src

Run a process and display output in the pile buffer:

#+begin_src emacs-lisp
(pile-make-process '("ls" "-la"))
#+end_src

The output appears in collapsible sections with status indicators showing whether the command succeeded (exit code 0) or failed.

* Customization

The package provides a =pile= customization group:

#+begin_src emacs-lisp
M-x customize-group RET pile RET
#+end_src

** Face Customization

- =pile-section-heading= - Inherits from =magit-section-heading= by default
